# Workflows

Procedimientos paso a paso para tareas repetitivas.

## Formato

Cada workflow es un archivo `.md` con esta estructura:

```markdown
# Workflow Name

## Description
What this workflow does and when to use it.

## Prerequisites
Required tools, access, or context.

## Steps
1. Step one
2. Step two
3. Step three

## Verification
How to confirm the workflow completed successfully.

## Rollback
Steps to undo if something goes wrong.
```

## Naming

Use action-oriented names: `deploy-service.md`, `create-release.md`, `onboard-project.md`
